Project Idea

Inventory Management System

Overview

I want to to be able to make a software solution which can be linked to POS system in a shop or a small restaurant and which will be able to record a businesses inventory and outgoing stock in an efficient manner, while inventory management systems exist for retail and warehouses, the usual practice of taking inventory in small businesses is manual and cumbersome.
Based on the pos sale data and ingredients used in a recipe , this system will be able to identify the stock levels of both perishable and non perishable ingredients in a restaurant.

Motivation

There are 85000 restaurants in Australia alone including takeaway and bars. Almost all of these restaurants still rely on manual stocktake using a pen and a paper which is performed by managers.
This is an inefficient and confusing process and almost always results in either ordering excessive stock which results in wastage or ordering less stock.
There can be upwards of 500 inventory items in a restaurant. For eg: An average house pantry alone may have more than 100 items. These not only include eatable items but items like packaging , coffee cups , takeaway boxes, freezer bags , day dots, cleaning chemicals etc.
There is a huge necessity to device an automated tool which is able to show the user its current stock levels and the projected levels that it may need from the suppliers.

Description

This tool has an immense capability to not only make the users life easier but it can facilitate in cutting down costs and wastage and will be able to show the user an accurate data of his/her usage to a certain degree of tolerances.

Working:

Lets take an example of your local pizza shop named Cheesy Pizza This system will take the input data from the user in terms of its menu and ingredients in its inventory.
Now it will ask the user for individual ingredients and their ratios used to make a particular recipe, as an example , each pizza that the restaurant sells can have 100gram of cheese on it and one pizza uses one pizza box, so the user will input this data into the system.
Based on the data in the inventory system and the sale data in the already existing pos system, the system is able to show the level of stock to the consumer and do the reminding of ordering as well.
Now a restaurant may have upwards of 20 suppliers, and its a labour intensive task to be able to keep the records of each suppliers delivery dates and invoices, Moreover restaurants are mostly cramped for space and they need constant renewal of stock unlike a warehouse.
This system will also be able to tell if a certain ingredient or a beverage is being consumed more or less and based on that a restaurant can advertise itself or change its business strategies depending upon the likes of patrons.

This system will reduce the food wastage in the kitchen substantially .

Tools and Technologies

This will be a software system will not need any hardware, It will simply be installed on the restaurants computer and would require integration with the its current pos system.
It would require Developer Technologies , Version Control System, Issue tracking and Documentation, Some examples of these tools are Java, Python, Git, UX/UI design, Microsoft tools, IntelliJ Idea, Stack Overflow , Jira. Testing tools such as Selenium or Protractor. Open Source tools can include editors like Eclipse when programming in Java.

Outcome

This software can be has an immense selling potential since there are a number of buyers. This will solve the problem of tons of food waste and tons of man hours wasted doing tedious task.
